AI coding agents are powerful but need guidance. Safeword provides portable hooks, skills, and quality controls that enforce TDD workflow and best practices across all your projects.

## Install

```bash
bunx safeword@latest setup
```

Or with npm: `npx safeword@latest setup`

## What You Get

| Path                | Description                     |
| ------------------- | ------------------------------- |
| `.safeword/`        | Hooks, guides, and templates    |
| `.claude/`          | Claude Code settings and skills |
| `.cursor/rules/`    | Cursor AI rules                 |
| `eslint.config.mjs` | ESLint with safeword plugin     |
| `AGENTS.md`         | Project context file            |

**Commit these to your repo** for team consistency.

## Verify

```bash
bunx safeword@latest check
```

## Start Coding

Open your project in Claude Code or Cursor. Hooks activate automatically:

- **SessionStart** — Version check, lint check, AGENTS.md verification
- **UserPromptSubmit** — Timestamp injection, question reminders
- **PostToolUse** — Auto-lint changed files
- **Stop** — Quality review prompt

## Commands

Quick actions via slash syntax:

| Command            | Purpose                             |
| ------------------ | ----------------------------------- |
| `/lint`            | Run linters and formatters          |
| `/audit`           | Architecture and dead code analysis |
| `/drift`           | Check if docs match reality         |
| `/quality-review`  | Deep code review with web research  |
| `/cleanup-zombies` | Kill zombie dev server processes    |

## Skills

Structured workflows that guide the AI:

| Skill                     | Purpose                                  |
| ------------------------- | ---------------------------------------- |
| `/safeword-debugging`     | Four-phase debugging (investigate first) |
| `/safeword-enforcing-tdd` | RED → GREEN → REFACTOR discipline        |
| `/safeword-refactoring`   | Small-step refactoring with tests        |
| `/safeword-brainstorming` | Collaborative design through questioning |
| `/safeword-writing-plans` | Create detailed execution plans          |

## Language Support

Auto-detected from your project:
