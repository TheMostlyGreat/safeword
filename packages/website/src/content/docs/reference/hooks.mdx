# Hooks

Hooks are TypeScript scripts that run at specific points during AI coding sessions. They execute with Bun runtime.

## Available Hooks

| Hook                       | Trigger          | Purpose                                  |
| -------------------------- | ---------------- | ---------------------------------------- |
| `session-version.ts`       | SessionStart     | Display safeword version                 |
| `session-verify-agents.ts` | SessionStart     | Verify AGENTS.md link exists             |
| `session-lint-check.ts`    | SessionStart     | Check for lint errors on session start   |
| `prompt-timestamp.ts`      | UserPromptSubmit | Inject timestamp into prompts            |
| `prompt-questions.ts`      | UserPromptSubmit | Remind agent to ask clarifying questions |
| `post-tool-lint.ts`        | PostToolUse      | Auto-lint after file edits               |
| `stop-quality.ts`          | Stop             | Quality review prompt on stop            |

## Hook Location

Hooks live in `.safeword/hooks/` and are executed with Bun:

```text
.safeword/hooks/
├── lib/
│   ├── lint.ts        # Multi-language lint logic
│   └── quality.ts     # Quality review helpers
├── post-tool-lint.ts
├── prompt-questions.ts
├── prompt-timestamp.ts
├── session-lint-check.ts
├── session-verify-agents.ts
├── session-version.ts
└── stop-quality.ts
```

## Multi-Language Linting

The `post-tool-lint.ts` hook routes files to the correct linter based on extension:

| Extension         | Linter        |
| ----------------- | ------------- |
| `.js`, `.ts`, etc | ESLint        |
| `.py`, `.pyi`     | Ruff          |
| `.go`             | golangci-lint |

Linters run silently if not installed (graceful fallback).

## Customizing Hooks

Hooks are configured in `.claude/settings.json` (see [Configuration](/reference/configuration#claude-settings)).

Hooks are "owned" files - they get overwritten on `safeword upgrade`. To customize:

1. Copy the hook to a new name (e.g., `my-custom-lint.ts`)
2. Modify as needed
3. Update `.claude/settings.json` to use your hook
4. Your custom hook won't be touched by upgrades
